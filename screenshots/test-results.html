🚀 NestJS Microservices — Rapport de Tests

Architecture REST · gRPC · Kafka · GraphQL — Tous les services opérationnels ✅

1
GET /products Catalog Service · Port 3000
Liste tous les produits stockés en SQLite via TypeORM
GET http://localhost:3000/products
📌 Le Catalog Service répond sur le port 3000 via REST. Les produits sont persistés en SQLite.







  

    
2

    

      
POST /products Créer un Produit

      
Ajout d'un nouveau produit "Smartphone Samsung" dans le catalogue

    

  

  
POST http://localhost:3000/products · Body: {"name":"Smartphone Samsung","price":800,"stock":15}

  
✅ Produit créé avec succès. L'ID est auto-incrémenté par TypeORM/SQLite.

  







  

    
3

    

      
POST /orders — Commande Valide gRPC + Kafka

      
Flux complet : Order → gRPC → Stock (décrement) → Kafka → Notification (email simulé)

    

  

  
POST http://localhost:3002/orders · Body: {"productId":1,"quantity":2,"customerEmail":"jean@dupont.com"}

  
✅ Commande confirmée. Kafka a déclenché le Notification Service → email envoyé à jean@dupont.com

  







  

    
4

    

      
POST /orders — Stock Insuffisant Validation gRPC ❌

      
Le Stock Service refuse via gRPC si la quantité dépasse le stock disponible

    

  

  
POST http://localhost:3002/orders · Body: {"productId":1,"quantity":100,"customerEmail":"test@test.com"}

  
❌ Comportement attendu : gRPC refuse la commande car quantité=100 > stock disponible. Commande non enregistrée.

  







  

    
5

    

      
GET /orders Order Service · Port 3002

      
Seules les commandes validées apparaissent — la commande refusée n'est pas enregistrée

    

  

  
GET http://localhost:3002/orders

  
📌 Seule la commande valide (étape 3) est enregistrée. La commande refusée (étape 4) n'apparaît pas.

  







  

    
6

    

      
GraphQL Dashboard Query Service · Port 3003

      
Agrégation products + orders en une seule requête GraphQL

    

  

  
⬡ http://localhost:3003/graphql

  
📌 Le Query Service agrège les données de plusieurs microservices en une seule requête GraphQL — Dashboard unifié.
